# Google AI Pro Tracker

## Project Overview

This project is a UI/UX prototype for a Google AI Pro subscription usage dashboard — see [README.md](README.md) for the important caveat. There is no real Google AI Pro usage API, so all usage statistics, analytics, and quota data are generated by a mock data layer (`js/api.js`) rather than a real integration. The dashboard, PWA shell, and Chrome Extension UI are real and functional; what they display is simulated. It is a feature-rich, single-page application (SPA) built with HTML, CSS, and vanilla JavaScript.

The application is structured with a clear separation of concerns, with dedicated JavaScript files for different functionalities like API interaction, storage, analytics, charts, and UI components. It also includes advanced features like ML-based analytics, PDF reporting, and enterprise-level features like team dashboards and Google Workspace integration.

## Building and Running

There are no explicit build steps or package management files (`package.json`, `requirements.txt`, etc.) in the project. The application can be run by opening the `index.html` file in a web browser.

**To run the application:**

1.  Open `index.html` in a modern web browser.

**To install the Chrome Extension:**

1.  Open Chrome and navigate to `chrome://extensions/`.
2.  Enable "Developer mode" in the top right corner.
3.  Click "Load unpacked" and select the `chrome-extension` directory from this project.
4.  The "AI Pro Tracker Extension" will be installed and ready to use.

## Development Conventions

*   **Technology Stack:** The project is built using HTML, CSS, and vanilla JavaScript. It uses Chart.js for creating charts.
*   **Code Style:** The JavaScript code is written using ES6 classes. There is a clear separation of concerns with different classes responsible for different functionalities.
*   **File Structure:** The project is organized into directories for CSS, JavaScript, icons, and other assets. The JavaScript directory is further divided into files for each major component of the application.
*   **Data Storage:** The application uses the browser's `localStorage` to store user data, managed by the `StorageService` class.
*   **API Integration:** The `APIManager` class handles interactions with a mock Google AI Pro API. This class is designed to be extended for real API integration.
*   **Modularity:** The application is highly modular, with different features encapsulated in their own JavaScript files (e.g., `analytics.js`, `goals.js`, `notifications.js`).
